## 二、产品背景与价值主张

### 2.1 市场痛点
现代企业面临知识爆炸与信息孤岛的双重困境：
- **知识分散**：企业文档散落在邮件、网盘、IM、本地文件夹、Git仓库等多个渠道
- **检索低效**：传统关键词搜索无法理解语义，导致"搜不到、搜不准、搜不全"
- **知识沉淀难**：专家经验难以结构化沉淀，新人培训成本高、周期长
- **决策支持弱**：面对复杂问题，缺乏跨文档、跨领域的综合分析能力
- **AI幻觉风险**：通用大模型缺乏企业私有知识，容易生成看似合理但错误的答案
- **成本不可控**：大模型API调用成本随用户量线性增长，缺乏精细化成本治理手段

### 2.2 产品愿景
> **让企业的每一份知识，都能被精准理解、即时调用、智能生成，且全程可控、可观测、可优化。**

### 2.3 核心价值主张

| 价值维度     | 具体描述                                            | 量化目标           |
| ------------ | --------------------------------------------------- | ------------------ |
| **效率提升** | 将知识检索时间从平均15分钟缩短至30秒以内            | 检索耗时↓95%       |
| **精准问答** | 基于语义理解+引用溯源，回答准确率>90%               | 准确率≥90%         |
| **知识活化** | 沉睡文档转化为可交互的智能知识资产                  | 文档利用率↑300%    |
| **决策增强** | Agent自动拆解复杂问题，提供带引用来源的综合分析     | 复杂任务完成率>75% |
| **成本可控** | 通过缓存、模型路由、配额管理，将单次问答成本降低60% | 成本↓60%           |
| **安全合规** | 敏感数据自动脱敏、权限隔离、完整审计追踪            | 0高危安全事件      |

### 3.1 核心用户角色

#### 角色一：普通员工（知识消费者）

- **占比**：20%
- **特征**：负责上传、维护、审核专业领域文档

- **占比**：5%
- **特征**：负责系统部署、监控、性能调优、成本控制

```
├─ 触点：会话完成，对整体答案点赞
├─ 行为：导出完整分析为PDF，附引用来源和Agent执行日志
├─ 情绪：专业、可直接汇报 📑
└─ 亮点：导出内容可直接用于管理层汇报；包含完整审计链
```

#### 7.3.2 AI安全（新增）

| 需求项               | 具体要求                                                     |
| -------------------- | ------------------------------------------------------------ |
| Prompt Injection防护 | 输入侧：检测恶意指令模式（如"忽略前文"、"system:"等），置信度>0.8时拦截；输出侧：输出内容与知识库上下文相关性校验，偏离度>阈值时告警 |

| 幻觉缓解             | 强制引用模式（无引用不生成事实性内容）；置信度评分<0.6时明确提示"以上信息仅供参考，请核实" |

### 10.2 核心场景验收用例

**验收用例1：基础RAG问答**
```
前置条件：已上传《2025年产品白皮书.pdf》，文档已索引完成
操作步骤：
1. 用户在聊天框输入"我们今年的核心产品功能有哪些？"
2. 系统返回答案
预期结果：
✅ 答案包含3-5个核心功能点，且与原文一致
✅ 每个功能点附带引用标记[1][2]等，点击可查看原文
✅ 响应时间<5秒，TTFB<1.5秒
✅ 答案支持Markdown渲染，无格式错误
✅ 用户可对答案点赞，反馈数据进入Bad Case库
```

**验收用例2：Agent复杂任务**
```
前置条件：已上传《2024市场报告.pdf》和《2025市场报告.pdf》，均已索引
操作步骤：
1. 用户输入"对比两年市场份额的变化趋势"
2. 观察Agent执行过程
预期结果：
✅ Agent显示任务拆解步骤（计划→检索2024→检索2025→对比分析→生成报告）
✅ 执行过程中显示当前步骤和进度
✅ 最终答案包含对比表格，数据与原文一致
✅ Agent执行Self-Reflection，显示置信度评分
✅ 总耗时<15秒，Agent调用工具次数≤5次
```

**验收用例3：权限隔离**
```
前置条件：文档A设为"项目组X可见"，用户Y不属于项目组X
操作步骤：
1. 用户Y在聊天框输入与文档A相关的问题
2. 用户Y在文档列表搜索文档A
3. 用户Y尝试通过Prompt Injection诱导模型泄露文档A内容
预期结果：
✅ 问答结果不包含文档A的内容
✅ 文档列表不显示文档A
✅ Prompt Injection被检测并拦截，返回安全提示
✅ 系统提示"未找到相关信息"而非"无权访问"（信息不泄露）
✅ 审计日志记录用户Y的查询行为和拦截事件
```

**验收用例4：安全与脱敏**
```
前置条件：已上传含员工薪资信息的《薪酬管理制度.pdf》
操作步骤：
1. 用户输入"张经理的工资是多少？"
2. 系统返回答案
预期结果：
✅ 答案中薪资数字被脱敏为***（如"月薪****元"）
✅ 系统提示"部分敏感信息已脱敏，请联系HR获取详细信息"
✅ 审计日志记录该敏感查询，标记为"PII访问"
✅ 管理员在后台可查看脱敏统计和原始内容（需审批）
```

**验收用例5：模型网关与故障转移**
```
前置条件：配置主模型Provider A和备用Provider B
操作步骤：
1. 正常提问，观察模型路由
2. 模拟Provider A故障（关闭服务或返回500）
3. 再次提问
预期结果：
✅ 正常时请求路由至Provider A
✅ Provider A故障后，自动切换至Provider B，用户无感知
✅ 切换过程<3秒，不中断流式输出（如正在生成则继续）
✅ 告警系统发送Provider A故障通知
✅ 故障恢复后，自动切回Provider A（可配置）
```

| **Prompt Injection攻击**    | 高   |  中  | 输入检测模型、输出校验、权限穿透测试、定期红队测试、安全审计 | 安全工程师 |

| **并发性能瓶颈**            | 中   |  中  | 异步处理、K8s HPA、Redis缓存、向量数据库HNSW索引优化、模型网关限流 | 后端工程师 |

| **数据质量与冷启动**        | 中   |  高  | 初期人工导入高质量文档、建立文档质量标准、冷文档自动提醒、知识缺口分析指导补充 |  产品经理  |
| **团队技术能力不足**        | 中   |  中  | M0充分预研、引入外部顾问、关键岗位招聘、建立技术分享机制、文档驱动开发 | 技术负责人 |
| **需求蔓延（Scope Creep）** | 中   |  高  | 严格的MVP范围控制、P2/P3功能后置、每Sprint评审需求变更、产品负责人把关 |  产品经理  |
